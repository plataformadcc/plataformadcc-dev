👾 Bem-vindo ao Ambiente DEV — PlataformaDCC

Este é o jogo de referência. Ele mostra exatamente o que você precisa fazer para submeter seu jogo. Leia com atenção — o processo é simples e direto.

1. Fork + Branch primeiro passo

  1. Acesse github.com/plataformadcc/plataformadcc-dev e clique em Fork.
  2. Clone o seu fork localmente: 
    git clone https://github.com/SEU-USUARIO/plataformadcc-dev.git
cd plataformadcc-dev
  3. Crie sua branch com o padrão obrigatório: 
    git checkout -b plataformadcc-[slug]-[seuNome]
    Exemplo: plataformadcc-space-shooter-joaosilva

2. Adicione seu jogo estrutura

Crie a pasta do seu jogo dentro de public/_games/:

public/_games/
└── seu-jogo/
    ├── index.html   ← entrypoint obrigatório
    ├── assets/      ← scripts, imagens, estilos
    └── README.md    ← objetivo pedagógico e controles

O slug é o nome da pasta: minúsculas, sem espaços, com hífens. Exemplo: multiplicacao-rapida.

3. Registre no catálogo obrigatório

Abra o arquivo games.config.ts na raiz do projeto e adicione seu jogo:

export const games = [
  // ... outros jogos
  {
    slug: 'seu-jogo',          // mesmo nome da pasta em public/_games/
    name: 'Nome do Jogo',
    description: 'Descrição curta do jogo.',
    compatibility: ['pc'],     // 'pc' | 'cell' | 'tablet'
  },
]

4. Teste localmente validação

npm install
npm run dev

Acesse http://localhost:3000 e confirme que seu jogo aparece no catálogo e roda corretamente em http://localhost:3000/games/seu-jogo.

⚠️ O jogo roda em iframe sandbox. Ele não tem acesso a cookies, localStorage da plataforma ou APIs de autenticação. Certifique-se de que seu jogo funciona nesse contexto antes de abrir o PR.

5. Abra o Pull Request envio

git add -A
git commit -m "feat: adiciona jogo seu-jogo"
git push origin plataformadcc-seu-jogo-seuNome

No GitHub, abra um Pull Request da sua branch para main no repositório plataformadcc/plataformadcc-dev.

✅ Checklist antes de abrir o PR

  • Pasta criada em public/_games/[slug]/
  • index.html presente e abrindo sem erros
  • Entrada adicionada em games.config.ts
  • Jogo funciona localmente em /games/[slug]
  • Jogo não depende de backend externo para rodar
  • Sem coleta de dados pessoais ou trackers
  • README.md dentro da pasta com objetivo e controles
  • Branch com padrão plataformadcc-[slug]-[seuNome]
